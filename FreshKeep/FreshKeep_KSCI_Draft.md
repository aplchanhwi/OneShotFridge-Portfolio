## 요약

본 논문에서는 가정 내 식재료의 등록 및 소비기한 관리 부담을 줄이기 위한 SwiftUI 기반 모바일 애플리케이션 원샷냉장고(FridgeShot)를 구현하였다. 원샷냉장고는 사용자가 식재료 이미지를 촬영하면 생성형 AI를 활용하여 이미지 내 식재료를 인식하고, 각 식재료의 권장 소비기한을 함께 제공한다. 사용자는 분석 결과를 확인한 뒤 식재료명과 소비기한을 수정하거나 직접 항목을 추가할 수 있으며, 저장된 데이터는 SwiftData를 통해 로컬에서 관리된다. 또한 소비기한 기반 알림과 보유 식재료 기반 레시피 추천 기능을 제공하여 식재료 관리와 활용 과정을 통합적으로 지원한다.

키워드: SwiftUI, SwiftData, 생성형 AI, 식재료 인식, 소비기한 관리

## I. 서론

가정에서 식재료를 관리할 때 사용자는 구매한 식재료의 종류와 소비기한을 지속적으로 기억해야 한다. 그러나 냉장고에 보관된 품목이 많아질수록 어떤 재료를 먼저 사용해야 하는지 파악하기 어렵고, 소비기한을 놓쳐 식재료가 폐기되는 문제가 발생한다. 기존의 식재료 관리 앱은 품목명과 날짜를 사용자가 직접 입력하는 방식이 많아 반복 사용에 부담이 있다.

본 논문에서는 이러한 문제를 해결하기 위해 SwiftUI 기반 식재료 관리 애플리케이션 원샷냉장고를 구현하였다. 원샷냉장고는 이미지 분석을 통해 식재료 후보와 권장 소비기한을 자동으로 제시하고, 사용자가 이를 검토한 뒤 저장할 수 있도록 설계되었다. 또한 소비기한 알림과 레시피 추천 기능을 함께 제공하여 식재료 등록, 관리, 활용 과정을 하나의 흐름으로 지원한다.

## II. 관련 연구

### 1. 모바일 식재료 관리

모바일 식재료 관리 서비스는 사용자가 보유 식재료를 기록하고 소비기한을 확인할 수 있도록 돕는다. 그러나 수동 입력 중심의 방식은 식재료가 많을수록 입력 과정이 번거롭고, 사용자가 소비기한을 직접 판단해야 한다는 한계가 있다. 따라서 식재료 등록 과정을 자동화하고 소비기한 정보를 함께 제공하는 기능이 필요하다.

### 2. 생성형 AI와 SwiftData

생성형 AI는 이미지와 텍스트를 기반으로 객체 인식 및 정보 추출을 수행할 수 있으며, 사용자의 입력 부담을 줄이는 데 활용될 수 있다. SwiftData는 Swift 객체 모델을 기반으로 데이터를 저장하고 조회할 수 있는 로컬 데이터 관리 프레임워크이다. 원샷냉장고는 생성형 AI를 통해 식재료명과 권장 소비기한을 추출하고, SwiftData를 활용하여 식재료 정보를 구조적으로 저장한다.

## III. 시스템 설계

![Fig. 1. 원샷냉장고 시스템 구성도 삽입 위치](figure-placeholder)

원샷냉장고는 사용자 인터페이스, AI 분석 서비스, 로컬 데이터 저장 계층으로 구성된다. 사용자는 SwiftUI로 구현된 화면에서 식재료 이미지를 촬영하거나 직접 식재료를 추가할 수 있다. 촬영된 이미지는 서버로 전송되어 식재료명과 권장 소비기한으로 분석되며, 앱은 분석 결과를 목록 형태로 표시한다. 사용자는 저장 전 결과를 수정하거나 삭제할 수 있으므로 AI 인식 오류에 대응할 수 있다.

로컬 데이터는 SwiftData 모델을 통해 관리된다. `FoodItem`은 사용자가 등록한 개별 식재료의 이름, 소비기한, 등록일, 소비 완료 여부를 관리하고, `Category`는 식재료별 기본 소비기한과 사용자 수정 소비기한을 저장한다. `CaptureSession`은 촬영 이미지와 인식 결과를 연결하여 이미지 기반 등록 이력을 관리한다. 이러한 구조를 통해 앱은 단순한 식재료 목록뿐 아니라 사용자 수정 정보와 촬영 기록을 함께 저장할 수 있다.

## IV. 구현 결과

### 1. 이미지 기반 식재료 등록

![Fig. 2. AI 식재료 분석 결과 화면 삽입 위치](figure-placeholder)

식재료 인식 기능은 사용자가 촬영하거나 선택한 이미지를 JPEG 형식으로 변환한 뒤 Base64 문자열로 인코딩하여 서버에 전달하는 방식으로 구현하였다. 서버 응답은 식재료명과 소비기한을 포함한 JSON 형식으로 반환되며, 앱은 이를 파싱하여 분석 결과 화면에 표시한다. 분석 결과 화면에서는 각 항목의 식재료명과 권장 소비기한을 확인할 수 있고, 사용자는 항목을 수정, 삭제, 추가한 뒤 냉장고 목록에 저장할 수 있다.

소비기한은 사용자가 직접 입력한 날짜를 우선 적용하고, 같은 식재료에 대해 사용자가 이전에 수정한 소비기한이 있으면 해당 값을 재사용한다. 기존 정보가 없는 식재료의 경우 AI가 제공한 권장 소비기한을 사용한다. 이를 통해 반복 입력을 줄이고 사용자의 실제 관리 기준을 반영할 수 있도록 하였다.

### 2. 소비기한 알림 및 레시피 추천

![Fig. 3. 냉장고 목록 및 레시피 추천 화면 삽입 위치](figure-placeholder)

저장된 식재료는 소비기한 날짜순으로 정렬되어 표시된다. 앱은 소비기한이 지난 경우, 오늘까지인 경우, 며칠 남은 경우를 구분하여 상태 문구를 제공한다. 또한 같은 날짜에 소비기한이 도래하는 식재료를 하나의 알림으로 묶어 예약함으로써 사용자가 만료 예정 식재료를 쉽게 확인할 수 있도록 하였다.

레시피 추천 기능은 사용자가 보유 식재료를 선택하고 난이도, 요리 스타일, 필수 재료, 기타 요구사항을 설정하면 AI가 적절한 레시피를 생성하는 방식으로 구현하였다. 생성된 레시피는 요리명, 조리 시간, 재료, 조리 단계로 구성되며, 앱 내부에 저장하여 다시 확인할 수 있다. 이를 통해 원샷냉장고는 식재료를 기록하는 기능을 넘어 남은 재료의 활용까지 지원한다.

## V. 결론

본 논문에서는 SwiftUI 기반 AI 식재료 관리 애플리케이션 원샷냉장고를 설계하고 구현하였다. 제안한 시스템은 이미지 분석을 통해 식재료명과 권장 소비기한을 자동으로 제시하고, 사용자가 검토한 결과를 SwiftData 기반 로컬 저장소에 관리하도록 구성하였다. 또한 소비기한 알림과 레시피 추천 기능을 제공하여 식재료 등록, 관리, 활용 과정을 통합적으로 지원하였다.

향후에는 이미지 인식 정확도에 대한 정량적 평가를 수행하고, 영수증 OCR 등록, 바코드 인식, 가족 간 냉장고 공유 기능을 추가하여 식재료 관리 편의성을 높일 계획이다.
