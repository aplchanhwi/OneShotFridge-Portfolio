# OneShotFridge Portfolio

FreshKeep는 냉장고 속 식재료를 더 쉽게 관리하기 위한 SwiftUI 기반 iOS 앱입니다.
사진으로 식재료를 등록하고, 소비기한을 관리하며, 보유한 식재료를 바탕으로 레시피 추천 흐름까지 이어지는 사용 경험을 목표로 만들었습니다.

이 저장소는 실제 서비스 중인 앱의 포트폴리오 공개용 버전입니다.
운영 API 주소, Firebase 설정 파일, 광고 식별자, StoreKit 상품 설정, 서명/푸시 권한 설정, 내부 실험 화면은 제거하거나 mock 구현으로 대체했습니다.

## 주요 기능

- SwiftUI 기반 냉장고 식재료 관리 화면
- SwiftData를 활용한 식재료, 카테고리, 촬영 기록, 레시피 모델링
- 식재료 등록 화면과 이미지 분석 결과 화면의 MVVM 분리
- 촬영한 이미지 분석 결과를 편집 가능한 식재료 목록으로 연결하는 흐름
- 선택한 식재료와 옵션을 바탕으로 한 레시피 추천 UI
- 소비기한 기반 로컬 알림 및 상태 관리 로직

## 성능 개선 기록

AI 이미지 분석 과정에서 원본 이미지를 그대로 전송하던 방식은 실제 기기 기준 평균 19.47초의 대기 시간이 발생했습니다.
이를 개선하기 위해 이미지 최대 변을 1280px로 제한하고 JPEG 품질을 0.65로 조정해, 평균 분석 시간을 10.10초로 약 48.1% 단축했습니다.

자세한 측정 조건과 정확도 trade-off는 [성능개선사항.md](./성능개선사항.md)에 정리했습니다.

## 공개 버전에서 제거한 항목

- 비공개 Firebase 설정 파일
- 실제 Cloud Functions 엔드포인트
- AdMob 앱 ID 및 광고 유닛 ID
- StoreKit 상품 설정 파일
- 앱 서명 entitlements 및 로컬 Xcode 사용자 상태 파일
- 내부 테스트/실험 화면

## 구조

```text
FreshKeep/
├── Back/
│   ├── Manager/      # 앱 상태, 저장소, 알림, 레시피 관리
│   ├── Service/      # 이미지 분석 및 추천 서비스 계층
│   └── Model.swift   # SwiftData 모델
├── Front/
│   ├── Recipe/       # 레시피 추천 관련 화면
│   └── ...           # 식재료 등록, 상세, 냉장고 화면
└── ViewModel/        # 화면별 상태와 비즈니스 로직 분리
```

## 참고

이 저장소의 일부 서비스 클래스는 실제 운영 서버 대신 placeholder 또는 mock 동작을 사용합니다.
목표는 운영 인프라를 노출하지 않으면서 앱 구조, SwiftUI 구현 방식, 제품 흐름 설계 과정을 보여주는 것입니다.
